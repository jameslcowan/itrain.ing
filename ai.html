powerlift.ing — AI assistant guide

This page exists so AI agents and search engines can quickly learn how to generate shareable powerlift.ing programs.

How powerlift.ing works

  • Stateless: no accounts; everything is client-only.
  • State-in-URL: the entire program is encoded into the share URL.
  • Routes: /p/<STATE> or /#/p/<STATE>.

Encoding contract (v1)

  1. Create a JSON object for the program state (schema below).
  2. JSON.stringify(state)
  3. Compress with lz-string: compressToUint8Array(json)
  4. Convert bytes to base64, then base64url-encode:
    • replace + with -, / with _
    • remove = padding
  5. Final link: https://powerlift.ing/p/<STATE>

Important constraint

If you cannot execute compression, do not output a fake /p/<STATE> URL. Instead: output the program + JSON + a short snippet that generates the real URL in a browser.

Program schema (v1)

{
  "v": 1,
  "u": "lb" | "kg",
  "c": [{ "n": "Meso 1" }],
  "weeks": [
    {
      "c": 0,
      "days": [
        {
          "label": "DAY 1 - MON",
          "rows": [
            { "ex": "", "mode": "", "sets": "", "reps": "", "load": "", "pct": "", "rpe": "", "rest": "" }
          ]
        }
      ]
    }
  ]
}

Defaults (if user doesn’t specify)

  • Units: kg
  • Frequency: 3 days/week
  • Duration: 4 weeks
  • Audience: general population
